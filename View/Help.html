前言

1. 所有 APP 产生的数据都保存在宿主机 /volume/AppName 中 .

2. 部分 APP 需要单独配置, 配置文件保存在 /volume/AppName/config 目录中 .

3. APP 产生的所有数据都不会被自动删除, 以便下次使用, 如果需要删除, 请登录宿主机手动删除 .

Redis

1. 过编辑按钮修改Redis配置 .

2. 初始密码为 123@456@ 如需修改,请修改配置项目 requirepass 123@456@ .

3. 启动服务后点击终端链接到一个装有 redis 客户端 的容器.

phpMyAdmin

1. 配置文件所在位置 App/phpMyAdmin/dependence/config.inc.php .

2. 参考文档 https://docs.phpmyadmin.net/zh_CN/latest/

Whistle

1. 通过设置网络代理, 来实现网络抓包功能 .

2. https 抓包请先安装证书, 证书通过点击页面中的 HTTPS 按钮得到 .

Memcached

1. memcached 的身份认证是通过 sasl 配置, 初始密码为 123@456@ ,可通过修改 App/Memcached/docker-entrypoint.sh 中的命令修改默认密码 .

2. 同时也可以通过 App/Memcached/docker-entrypoint.sh 中的命令来修改启动参数 .

ResilioSync

1. 创建新的同步项时, 请把同步目录选择到 /volume 的子目录下, 以便数据持久化 .

2. 请通过 设置 -> 高级 -> 链接 -> 侦听端口 修改端口号为 55555 以便开启局域网加速 .

VerySync

1. 创建新的同步项时, 请把同步目录选择到 /volume 的子目录下, 以便数据持久化 .

MySQL 5.7

1. 数据库安装成功后, 请务必第一时间通过Web管理工具创建管理用户密码 .

DokuWiki

1. 安装成功后,访问服务 IP:Port/install.php 来安装 DokuWiki 配置 用户名,密码 .

PHP72

1. 安装成功后,通过 /volume/PHP72/config 来修改 PHP, Nginx 配置文件 .

2. 通过 /volume/PHP72/workdir 目录来部署 PHP 代码 .

Net2Ftp

1. 安装成功后,通过 /volume/Net2Ftp/config 来修改 PHP, Nginx 配置文件 .

2. 通过 /volume/Net2Ftp/workdir 目录来修改 settings.inc.php 等 Net2Ftp 配置文件 .

3. 关于 Net2Ftp 的上传, 下载限制, 日志等配置请参考官方文档

Shadowsocks

1. 可通过编辑按钮修改服务配置文件 .

2. 配置代理的时候请注意 "加密方式", 默认为 aes-256-cfb .

NextCloud

1. 此服务需要 MySQL/MariaDB 数据库支持.

2. 登录后点击右上角的 用户头像->关于 下载客户端使用.

3. 强制使用 HTTPS 连接, chrome 无法信任, 请使用 /Volumes/DockerStore/WebSSH/workdir/ssl/ 目录中的根证书, 导入至系统.

4. 修改配置文件开启图片与视频处理插件.

    'enable_previews' => true,

    'enabledPreviewProviders' =>

        array (

            0 => 'OC\\Preview\\Image',

            1 => 'OC\\Preview\\Movie',

            2 => 'OC\\Preview\\TXT',

    ),

Frp Server

1. 通过修改 frps.ini 来修改 Frp Server 的服务配置 .

2. 由于 DockerStore 限制, 开放端口必须为 9500-9599 .

3. 默认用户名: admin, 默认密码: 123@456@

Frp Server

1. 通过修改 frpc.ini 来修改 Frp Client 的服务配置 .

2. 示例配置文件在 frpc_example.ini , 复制过来使用即可 .

3. 默认用户名: admin, 默认密码: 123@456@

Web SSH

1. 强制使用 HTTPS 连接, chrome 无法信任, 请使用 /Volumes/DockerStore/WebSSH/workdir/ssl/ 目录中的根证书, 导入至系统.

2. 系统每次登陆会修改密码, 所以链接不能重复使用.

Aria2

1. Web UI 默认用户名: admin, 密码: 123@456@.

2. 通过 htpasswd -bc /volume/config/htpasswd hello 123456 创建用户.

3. 通过 htpasswd -D /volume/config/htpasswd admin 删除用户.

4. 通过 htpasswd -b /volume/config/htpasswd admin 123456 修改密码.

5. JSON-RPC 链接的配置示例 https://token:qRc32PlTRI72wQSG7T0R8yR1QxXYRSk4@127.0.0.1:6800/jsonrpc

6. 默认 secret 请通过"编辑"功能查看. 配置文件: /volume/config/aria2.conf 搜索 rpc-secret .

2. 增加 redis_servers 中的字段, 增加服务器.

3. Web UI 默认用户名: admin, 密码: 123@456@.

4. 通过 htpasswd -bc /volume/config/htpasswd hello 123456 创建WebUi用户.

5. 通过 htpasswd -D /volume/config/htpasswd admin 删除WebUi用户.

6. 通过 htpasswd -b /volume/config/htpasswd admin 123456 修改WebUi密码.

CoreDNS

1. CoreDNS是一个DNS服务器/转发器 .

2. 部分宿主机系统部署可能不能启动此服务,需要关闭宿主机的 systemd-resolved 服务 (service systemd-resolved stop), 并重新配置 resolv.conf 请谨慎操作 .

3. 如果宿主机不能上网, 服务启动后需要把 resolv.conf 中的 nameserver 地址改为 127.0.0.1 .

4. 官方文档,请点击 .

OpenResty

1. 可以通过编辑功能清空/workdir/nginx/conf/nginx.conf文件中的内容释放 0.0.0.0 ip监控 .

2. /Volumes/DockerStore/OpenResty/nginx 为 nginx 配置文件目录.

3. /Volumes/DockerStore/OpenResty/conf.d 为 自定义配置文件目录.

4. /Volumes/DockerStore/OpenResty/lualib 为 自定义模块包目录.

5. /Volumes/DockerStore/OpenResty/workdir 为 自定义 lua 文件目录.

Emby

1. 由于项目全部修改为HTTPS, 需执行部分操作才能使用 Emby .

2. Emby 启动后, 点击 Emby App 的终端按钮 .

sed -i "5i\ \ \<CertificatePath>/etc/nginx/ssl/iis-dockerstore.ssh2.cc.pfx</CertificatePath>" /config/config/system.xml

head -1 /etc/nginx/ssl/iis-pfx-password.txt | tr "\n" " " | xargs -i sed -i "6i\ \ \<CertificatePassword>{}</CertificatePassword>" /config/config/system.xml

3. 完成后, 击 Emby App 的重启按钮即可, 根据机器性能情况, 重启可能需要一段时间, 请耐心等待.

MinIO

1. 默认 AccessKey: minioadmin , SecretKey: minioadmin .

2. 首次修改密码, 通过编辑配置文件 /volume/.env 增加如下配置即可
- AccessKey: MINIO_ACCESS_KEY=admin
- SecretKey: MINIO_SECRET_KEY=123@456@

3. 以后修改密码, 通过编辑配置文件 /volume/.env 参考: 使用新凭修改加密 通过增加, MINIO_ACCESS_KEY_OLD, MINIO_SECRET_KEY_OLD 进行修改.

TdDownload

2. 在日志中搜索 THE ACTIVE CODE IS, 复制冒号后的 Code .
